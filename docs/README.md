# Documentation Index

All project guidance lives inside the `docs/` tree to keep the repo root clean. Key groups:

- **Product & flows**: `prd.md`, `design.md`, `job_flow.md`, `vehicle_flow.md`, `user_profile_flow.md`, etc.
- **Operational guides**: `DEPLOYMENT_GUIDE.md`, `supabase_dashboard_settings.md`, `FCM_MULTI_DEVICE_SETUP.md`, `FCM_V1_SETUP.md`, `PUSH_NOTIFICATIONS*.md`.
- **Audits & research**: `docs/audits/AUDIT_REPORT.md`, `docs/audits/SAFE_CLEANUP_PLAN.md`, `ANDROID_NOTIFICATION_AUDIT.md`, `OPTION_A_IMPLEMENTATION_SUMMARY.md`.
- **Data & architecture**: `data_model.md`, `choice-lux-cars.md`, `CLIENT_*` docs, `driver_*` docs, `VOUCHER_FLOW.md`, `QUOTE_FLOW.md`.

Add new docs here and update this index so teammates can discover the latest references quickly. If a file is environment-specific (e.g., Supabase settings, Firebase instructions), place it alongside the related guide rather than the repo root.
